## プロセス管理

サーバー上で実行中のプロセスを表示および管理します。

### プロセスリスト

- すべての実行中のプロセスの詳細
- PID（プロセス ID）
- CPU およびメモリ使用率
- ユーザー所有権
- プロセスコマンド

### プロセスアクション

- **強制終了 (Kill)**: プロセスを終了します
- **フィルター**: 名前またはユーザーで絞り込み
- **ソート**: CPU、メモリ、または PID で並べ替え
- **検索**: 特定のプロセスを検索

## Systemd サービス

サービス制御のために systemd サービスを管理します。

### サービスリスト

- すべての systemd サービス
- アクティブ/非アクティブ状態
- 有効/無効状態
- サービスの説明

### サービスアクション

- **開始**: 停止中のサービスを起動
- **停止**: 実行中のサービスを停止
- **再起動**: サービスを再起動
- **有効化**: 起動時の自動開始を有効化
- **無効化**: 自動開始を無効化
- **ステータスを表示**: サービスのステータスとログを確認
- **リロード**: サービスの構成をリロード

## 要件

- SSH ユーザーに適切な権限が必要です
- サービス管理の場合: `sudo` アクセスが必要になる場合があります
- プロセス表示: 通常、標準ユーザー権限で十分です

## ヒント

- プロセスリストを使用して、リソースを大量に消費しているプロセスを特定します
- トラブルシューティングのためにサービスログを確認します
- 自動更新を使用して重要なサービスを監視します
