Flutter Server Box は、Proxmox VE 仮想化プラットフォームの管理をサポートしています。

## 機能

### VM 管理

- **VM 一覧**: すべての仮想マシンを表示
- **VM ステータス**: 実行/停止状態を確認
- **VM アクション**: VM の開始、停止、再起動
- **VM 詳細**: 構成とリソースの表示

### コンテナ (LXC) 管理

- **コンテナ一覧**: すべての LXC コンテナを表示
- **コンテナステータス**: コンテナの状態を監視
- **コンテナアクション**: コンテナの開始、停止、再起動
- **コンソールアクセス**: コンテナへのターミナルアクセス

### ノード監視

- **リソース使用状況**: CPU、メモリ、ディスク、ネットワーク
- **ノードステータス**: ノードの健全性を確認
- **クラスタービュー**: マルチノードクラスターの概要

## セットアップ

### PVE サーバーの追加

1. 通常の SSH 接続としてサーバーを追加します
2. ユーザーが PVE 権限を持っていることを確認します
3. サーバー詳細ページから PVE 機能にアクセスします

### 必要な権限

PVE ユーザーには以下が必要です：

- **VM.Audit**: VM ステータスの表示
- **VM.PowerMgmt**: VM の開始/停止
- **VM.Console**: コンソールアクセス

権限設定の例：

```bash
pveum useradd myuser -password mypass
pveum aclmod /vms -user myuser@pve -role VMAdmin
```

## 使用方法

### VM 管理

1. PVE を搭載したサーバーを開く
2. **PVE** ボタンをタップ
3. VM リストを表示
4. VM をタップして詳細を表示
5. 管理用のアクションボタンを使用

### コンテナ管理

1. PVE を搭載したサーバーを開く
2. **PVE** ボタンをタップ
3. [コンテナ] タブに切り替える
4. LXC コンテナを表示して管理

### 監視

- リアルタイムのリソース使用状況
- 履歴データのチャート
- 複数ノードのサポート

## ステータス別の機能

### 実装済み

- VM の一覧表示とステータス
- コンテナの一覧表示とステータス
- 基本的な VM 操作（開始/停止/再起動）
- リソース監視

### 計画中

- テンプレートからの VM 作成
- スナップショット管理
- コンソールアクセス
- ストレージ管理
- ネットワーク構成

## 要件

- **PVE バージョン**: 6.x または 7.x
- **アクセス**: PVE ホストへの SSH アクセス
- **権限**: 適切な PVE ユーザーロール
- **ネットワーク**: PVE API への接続性（SSH 経由）

## ヒント

- 権限を制限した **専用の PVE ユーザー** を使用してください
- 最適なパフォーマンスのために **リソース使用状況** を監視してください
- メンテナンスの前に **VM ステータス** を確認してください
- 重大な変更の前に **スナップショット** を使用してください
